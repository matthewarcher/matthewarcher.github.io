Jet Coordinates

 Three of my publications utilize code I wrote during my PhD that transforms 3D velocity data (x, y, t) of a meandering jet from geographical (longitude/latitude) to jet coordinates (cross-stream/alongstream). This new frame of reference makes the maximum velocity core the origin, rather than a geographical location, and reshuffles gridpoints to follow the jet shape/orientation. This greatly improves the time-mean representation of the jet; the sharp velocity and lateral shear of the jet is retained in the time mean, allowing for a better representation of the fluctuating component (more details here).

I am currently working on making this Matlab code available on GitHub in a more generalized form. In the meantime, feel free to contact me for more details.

Python

 I am (slowly!) transitioning to Python. There are some really useful resources for oceanographers (and non-oceanographers alike) online:
  • To begin, you need to set up your environment. I highly recommend Anaconda, which makes it so easy (or Miniconda, which is the 'lighter' version). Anaconda also comes with a package manager (Conda), making it easy to install and update additional packages.

  • Anaconda also includes several work space options, including: Jupyter Notebook (best for online coding, teaching, or sharing); Spyder (if you can't let go of the Matlab-style GUI interface); IPython Console.

  • Data Carpentry provides free, easy-to-follow online workshops. A great hands-on starter course is this one. With a little bit of experience, a perfect course for oceanographers is run by Damien Irving here.
